Github Terms

  • Repository

    • A repository contains all of the project files (including documentation), and stores each file's revision history. Repositories can have multiple collaborators and can be either public or private.

  • Git Ignore

    • pending

  • Clone

    • A copy of a repository that resides in your computer rather than online.

  • Fork

    • A personal copy of another user's repository that lives on your account. It allows you to freely make changes to a project without affecting the original.

  • Branch

    • A copy version of a repository that does not affect the master branch and allows you to code freely without disrupting your original version.

  • Commit

    • It is a change to a file or multiple files, that includes a brief description of what the changes to the files were made

  • Merge

    • Takes the changes from one branch (in the same repository or from a fork), and applies them into another. I can be done automatically via a pull request via the GitHub web interface.

  • Checkout

    • pending

  • Push

    • Sending your committed changes to a remote repository, such as a repository hosted on GitHub.

  • Pull

    • When you are fetching changes and merging them.

  • Master Branch

    • This is your default branch that is automatically given to you when you create a repository and is considered a definitive branch.
